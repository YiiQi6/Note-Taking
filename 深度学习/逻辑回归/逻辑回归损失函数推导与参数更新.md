# 逻辑回归：损失函数推导与参数更新

> 逻辑回归名字里带"回归"，但它做的是分类。理解它的关键在于——损失函数不是随便定义的，而是从概率推导出来的。

---

## 一、为什么不能复用 MSE？

### 1.1 逻辑回归做什么

线性回归：`ŷ = wx + b`  →  输出任意实数，预测连续值

逻辑回归：`ŷ = σ(wx + b)`  →  输出 0~1 之间的数，理解为概率

```
          1
σ(z) = ———————    z = wx + b
       1 + e⁻ᶻ
```

```
sigmoid 的样子：
  1 ┤        ╱━━━━━━
    │       ╱         ← z 很大时，输出趋近 1
0.5 ┤    ╱
    │   ╱              ← z=0 时，输出 0.5
  0 ┼━━━━━━╱
    └─────────────→ z  ← z 很小时，输出趋近 0
```

### 1.2 如果硬套 MSE 会怎样

```
loss = (σ(wx+b) - y)²

结果：非凸函数，满地局部最小值，梯度下降会卡住 → 不能用
```

## 二、损失函数推导（核心）

### 2.1 第一步：把输出解释为概率

```
ŷ = P(y=1 | x)     ← 给定特征 x，标签为 1 的概率
1-ŷ = P(y=0 | x)   ← 给定特征 x，标签为 0 的概率
```

### 2.2 第二步：用一个式子统一 y=0 和 y=1

伯努利分布的概率质量函数：

```
P(y) = ŷ^y · (1-ŷ)^(1-y)
```

验证：

| 真实 y | 公式代入        | 结果             |
| ---- | ----------- | -------------- |
| y=1  | ŷ¹ · (1-ŷ)⁰ | ŷ（预测为 1 的概率）   |
| y=0  | ŷ⁰ · (1-ŷ)¹ | 1-ŷ（预测为 0 的概率） |

**这个 P(y) 的含义是：在当前模型参数下，看到这个样本的可能性（似然）。**

### 2.3 第三步：所有样本的联合似然

m 个独立样本一起出现的概率 = 各自概率的乘积：

```
L = P(y₁) · P(y₂) · ... · P(yₘ)
  = ∏ ŷᵢ^yᵢ · (1-ŷᵢ)^(1-yᵢ)
```

**最大似然估计（MLE）的思想**：找参数 w、b，让"观察到当前数据"这件事的概率最大。

### 2.4 第四步：取对数（乘法 → 加法）

m 个 (0,1) 之间的数相乘，结果极小，计算机直接溢出。取对数解决：

```
log L = Σ [ yᵢ·log(ŷᵢ) + (1-yᵢ)·log(1-ŷᵢ) ]
```

对数函数是单调递增的，原来使 L 最大的参数，同样使 log L 最大。

### 2.5 第五步：最大化 → 最小化

机器学习习惯最小化损失，所以加负号，再除以 m：

```
          1    m
loss = - ———   Σ  [ yᵢ·log(ŷᵢ) + (1-yᵢ)·log(1-ŷᵢ) ]
          m   i=1
```

这就是**二分类交叉熵损失（Binary Cross-Entropy Loss）**。

### 2.6 直观理解：它到底在罚什么

| 真实标签 | 损失        | 预测越准        | 预测越离谱        |
| ---- | --------- | ----------- | ------------ |
| y=1  | -log(ŷ)   | ŷ→1, loss→0 | ŷ→0, loss→+∞ |
| y=0  | -log(1-ŷ) | ŷ→0, loss→0 | ŷ→1, loss→+∞ |

```
y=1 时：                        y=0 时：
loss ↑                          loss ↑
 ∞ ┤●                             ∞ ┤            ●
   │╲                               │           ╱
   │ ╲                              │          ╱
   │  ╲                             │         ╱
 0 ┤   ●━━                          │   ━━●╱
   └───────→ ŷ                       └───────→ ŷ
   0      1                          0      1

预测错误时惩罚呈指数级增长 → 模型不敢瞎猜
```

## 三、深入理解"似然"——这可能是全文最重要的一节

### 3.1 概率 vs 似然：硬币的例子

**概率**：参数已知，问你某件事发生的可能性。

> 一枚硬币正面概率是 0.5，抛 10 次出现 7 次正面的概率是多少？
> → 参数（0.5）已知，问结果。这是**概率**。

**似然**：结果已知，问你参数是多少。

> 一枚硬币抛了 10 次，出现 7 次正面。这枚硬币正面概率最有可能是多少？
> → 结果（7次正面）已知，反推参数。这就是**似然**。

一句话：

|     | 已知  | 问         | 方向      |
| --- | --- | --------- | ------- |
| 概率  | 参数  | 结果发生的可能   | 参数 → 结果 |
| 似然  | 结果  | 什么样的参数最合理 | 结果 → 参数 |

### 3.2 用逻辑回归的场景翻译

你的数据集是：

```
样本1：x=1,  y=1（这封邮件是垃圾邮件）
样本2：x=2,  y=0（这封邮件不是垃圾邮件）
样本3：x=3,  y=1（这封邮件是垃圾邮件）
```

拿到这份数据后，你要反推出模型参数 w 和 b。

核心问题变成：

> 什么样的 w 和 b，能让模型"看到"这 3 个样本的时候，觉得它们最**不意外**？

这就是似然的思考方式——**用观察到的数据，去反向评估参数的好坏**。

### 3.3 "观测到样本的概率"到底是什么？

假设某组参数下，模型的预测情况是：

```
样本1（真的垃圾邮件）：ŷ₁ = 0.9  → 模型说"90%是垃圾邮件"
样本2（不是垃圾邮件）：ŷ₂ = 0.2  → 模型说"20%是垃圾邮件"
样本3（真的垃圾邮件）：ŷ₃ = 0.8  → 模型说"80%是垃圾邮件"
```

那么模型"看到"这个数据集的可能性：

```
样本1 标签为 1，出现的概率 = ŷ₁ = 0.9
样本2 标签为 0，出现的概率 = 1-ŷ₂ = 0.8
样本3 标签为 1，出现的概率 = ŷ₃ = 0.8

全部一起出现 = 0.9 × 0.8 × 0.8 = 0.576
```

换一组烂参数再算：

```
样本1：ŷ₁ = 0.3（明明是垃圾邮件，却只给 30%）
样本2：ŷ₂ = 0.7（明明不是垃圾邮件，却给 70%）
样本3：ŷ₃ = 0.1（明明是垃圾邮件，却只给 10%）

全部一起出现 = 0.3 × 0.3 × 0.9 = 0.081
```

同样一份数据：

- 好参数下，数据出现的似然是 **0.576**
- 烂参数下，数据出现的似然是 **0.081**

**我们希望找到使 0.576（最大值）的那组参数，而非 0.081。**

### 3.4 一个能从根本上扭转直觉的类比

你去一家奶茶店，连续 3 天观察到：

```
第一天：顾客买了（y=1）
第二天：顾客没买（y=0）
第三天：顾客买了（y=1）
```

你想知道"这位顾客每天来买的概率 p"是多少。

你试两个猜测：

**猜测 A：p = 0.9**（顾客 90% 会买）

```
第1天买   的概率：0.9
第2天没买 的概率：1-0.9 = 0.1
第3天买   的概率：0.9

似然 = 0.9 × 0.1 × 0.9 = 0.081
```

**猜测 B：p = 0.6**（顾客 60% 会买）

```
第1天买   的概率：0.6
第2天没买 的概率：1-0.6 = 0.4
第3天买   的概率：0.6

似然 = 0.6 × 0.4 × 0.6 = 0.144
```

> 猜测 B 的似然是 0.144，大于猜测 A 的 0.081。说明 p=0.6 比 p=0.9 更能解释你观察到的数据。

最大似然估计就是：不是试两下，而是**直接算出使似然最大的那个 p**。

### 3.5 为什么要取对数？

两个原因：

**原因一：防止数值溢出。**

1000 个样本，每个概率在 (0,1)，乘完的结果小到计算机无法表示。

```
0.5¹⁰⁰⁰ ≈ 10⁻³⁰¹  →  连 double 都装不下
```

取对数后：

```
log(0.5) × 1000 ≈ -693  →  干净利落
```

**原因二：乘法变加法，求导更方便。**

```
求 L = a × b × c 的最大值  →  需要求积的导数，很麻烦
求 log L = log a + log b + log c 的最大值  →  逐项求导再相加，简单
```

而取对数不影响结果——因为 `log(x)` 是单调递增的：

```
如果 L(A) > L(B)，那么 log L(A) > log L(B)

最大化 L 和最大化 log L，得到的最优参数完全一样
```

### 3.6 整个推导的直觉串起来

```
我有 m 个样本，标签非 0 即 1
        ↓
每个样本的出现概率可以统一写成一个式子：ŷ^y · (1-ŷ)^(1-y)
        ↓
所有样本一起出现的概率 = 全部相乘（联合似然）
        ↓
我要求：让这个联合似然最大的参数 ← 这就是 MLE
        ↓
取对数（方便计算 + 防止溢出，不影响答案）
        ↓
加负号（最大化变成最小化，方便梯度下降）
        ↓
得到交叉熵损失
```

> **似然，本质上就是"在某个参数设定下，当前这组数据看起来有多合理"。参数越好，数据就越"意料之中"，似然就越大。**

## 四、参数更新：梯度怎么算

### 4.1 损失函数对 w 求偏导

推导要用链式法则，分三步：

```
loss 对 w 的偏导 = ∂loss/∂ŷ  ·  ∂ŷ/∂z  ·  ∂z/∂w
                    └─┬─┘     └─┬─┘    └─┬─┘
                   第1层      第2层     第3层
```

**第1层：损失对预测值求导**

```
loss = -[y·log(ŷ) + (1-y)·log(1-ŷ)]

∂loss/∂ŷ = -y/ŷ + (1-y)/(1-ŷ) = (ŷ - y) / (ŷ(1-ŷ))
```

**第2层：预测值对 z 求导（sigmoid 的导数）**

```
ŷ = 1/(1+e⁻ᶻ)

∂ŷ/∂z = ŷ(1-ŷ)    ← sigmoid 的神奇性质，导数是自身乘(1-自身)
```

**第3层：z 对 w 求导**

```
z = wx + b

∂z/∂w = x
```

**三层相乘：**

```
∂loss/∂w = ∂loss/∂ŷ · ∂ŷ/∂z · ∂z/∂w

         = (ŷ - y) / (ŷ(1-ŷ)) · ŷ(1-ŷ) · x
              ↑                      ↑
              正好消掉！

         = x · (ŷ - y)
```

最终结果**简洁到惊人**：

```
∂loss/∂w = x · (ŷ - y)
```

### 4.2 损失函数对 b 求偏导

同理，唯一区别是 ∂z/∂b = 1：

```
∂loss/∂b = ŷ - y
```

### 3.3 和线性回归的梯度对比

|          | 线性回归         | 逻辑回归          |
| -------- | ------------ | ------------- |
| 模型       | ŷ = wx + b   | ŷ = σ(wx + b) |
| 损失函数     | MSE          | 交叉熵           |
| ∂loss/∂w | `2x·(ŷ - y)` | `x·(ŷ - y)`   |
| ∂loss/∂b | `2·(ŷ - y)`  | `ŷ - y`       |

**形式几乎一样！** 区别只是逻辑回归没有那个系数 2（来自 sigmoid 求导时恰好消掉了分母），且 ŷ 经过了 sigmoid。

> 为什么梯度形式这么像？不是巧合——交叉熵 + sigmoid 的组合恰好把中间繁杂的导数消掉，留下了干净的形式。这是数学上精心搭配的结果。

### 3.4 参数更新公式

```
w_new = w_old - α · (1/m) · Σ xᵢ · (ŷᵢ - yᵢ)
b_new = b_old - α · (1/m) · Σ (ŷᵢ - yᵢ)
```

其中 α 是学习率，m 是样本数。

### 3.5 用代码来感受

```python
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# 数据：x₁=1,y=1  x₂=2,y=0  x₃=3,y=1
X = np.array([1, 2, 3])
y = np.array([1, 0, 1])
w, b = 0.0, 0.0
lr = 0.1

for epoch in range(100):
    z = w * X + b
    y_pred = sigmoid(z)                 # 前向传播

    dw = np.mean(X * (y_pred - y))      # ∂loss/∂w，所有样本取平均
    db = np.mean(y_pred - y)            # ∂loss/∂b

    w -= lr * dw                         # 梯度下降
    b -= lr * db

    loss = -np.mean(y * np.log(y_pred + 1e-8)
                    + (1-y) * np.log(1 - y_pred + 1e-8))
    # +1e-8 防止 log(0)
```

### 3.6 为什么用交叉熵而不是 MSE：梯度的角度看

如果用 MSE（逻辑回归里偶尔也敢用，但要看情况）：

```
loss_mse = (ŷ - y)²

∂loss/∂w = 2(ŷ - y) · ∂ŷ/∂z · x
         = 2(ŷ - y) · ŷ(1-ŷ) · x
                          ↑
                    这个 ŷ(1-ŷ) 是个坑！
```

当 ŷ 接近 0 或 1（预测很确信，但可能完全错了）：

```
ŷ ≈ 0：ŷ(1-ŷ) ≈ 0 → 梯度 ≈ 0 → 卡住不动！
ŷ ≈ 1：ŷ(1-ŷ) ≈ 0 → 梯度 ≈ 0 → 卡住不动！
```

这就是**梯度消失**：预测越离谱，按理应该步子越大去纠错，但 MSE 恰恰相反——步子几乎为 0，训不动。

交叉熵正好把这个 `ŷ(1-ŷ)` 消掉了，梯度直接正比于误差 `(ŷ-y)`，预测越离谱，梯度越大，步子越大——这才是合理的行为。

## 四、损失函数推导路线图

```
假设标签服从伯努利分布
        ↓
写出单个样本的概率：ŷ^y · (1-ŷ)^(1-y)
        ↓
所有样本联合似然：全部相乘
        ↓
取对数：乘法变加法（不改变最优点位置）
        ↓
加负号，除以 m：最大化 → 最小化，同时与样本数脱钩
        ↓
得到交叉熵损失 ✅
        ↓
求导时：交叉熵 × sigmoid → 中间项恰好消掉
        ↓
梯度 = x·(ŷ - y)，简洁到像故意设计的一样
```

## 五、模型评估报告解读

训练完模型之后，你需要知道它到底好不好。下面这份报告是用 sklearn 的 `classification_report` 生成的：

```
              precision    recall  f1-score   support

          良性       0.93      0.95      0.94        41
          恶性       0.97      0.96      0.97        73

    accuracy                           0.96       114
   macro avg       0.95      0.96      0.95       114
weighted avg       0.96      0.96      0.96       114
```

### 5.1 先看混淆矩阵——所有指标的源头

模型预测结果只有两种情况：预测对了、预测错了。把真实标签和预测结果交叉，就得到混淆矩阵：

|                | 预测为恶性 | 预测为良性 |
| -------------- | ----- | ----- |
| **真实为恶性**     | TP    | FN    |
| **真实为良性**     | FP    | TN    |

四个格子：

- **TP（True Positive）**：真的是恶性，也预测对了 → 73 × 0.96 ≈ 70 个
- **FN（False Negative）**：真的是恶性，却漏掉了 → 73 - 70 ≈ 3 个
- **FP（False Positive）**：明明是良性，却误判为恶性 → 41 × (1-0.95) ≈ 2 个
- **TN（True Negative）**：真的是良性，也预测对了 → 41 × 0.95 ≈ 39 个

> 你的模型在 114 个样本中，大约判错 5 个（3 个恶性被漏掉 + 2 个良性被误判），其余全对。

### 5.2 Precision（精确率 / 查准率）

```
                 TP
Precision = ————————————
              TP + FP
```

**含义**：模型说"恶性"的所有样本中，有多少真的是恶性。

**你的良性 Precision = 0.93**：模型说"这是良性"时，有 93% 的概率没说错。
**你的恶性 Precision = 0.97**：模型说"这是恶性"时，有 97% 的概率说对了。

直观类比：Precision 衡量的是**"宁可放过，不可错杀"**——你说出来的话，要对多少。

什么时候优先看 Precision？垃圾邮件过滤——你把重要邮件判成垃圾邮件的代价很高，所以要求 Precision 高。

### 5.3 Recall（召回率 / 查全率）

```
                 TP
Recall = ————————————
              TP + FN
```

**含义**：所有真实为恶性的样本中，有多少被模型揪出来了。

**你的良性 Recall = 0.95**：100 个良性患者中，模型找出了 95 个。
**你的恶性 Recall = 0.96**：100 个恶性患者中，模型找出了 96 个。

直观类比：Recall 衡量的是**"宁可错杀，不可放过"**——该抓的，你抓到了多少。

什么时候优先看 Recall？癌症筛查——漏掉一个恶性患者的代价远远大于把良性误判为恶性，所以要求 Recall 高。

### 5.4 Precision 和 Recall 的跷跷板关系

这两个指标天然互相制约：

```
阈值调低（更容易判恶性）：
  → 更容易抓到恶性，Recall ↑
  → 但也更容易把良性误判为恶性，Precision ↓

阈值调高（更难判恶性）：
  → 被误判的良性少了，Precision ↑
  → 但有些不太明显的恶性被漏掉了，Recall ↓
```

```
        高阈值 ←——————————————————→ 低阈值

Precision  ████████████████░░░░░░░░  高 → 低
Recall     ░░░░░░░░░░░█████████████  低 → 高
```

你这份报告里 Precision 和 Recall 都很高且很均衡，说明阈值选得合适、模型本身也好。

### 5.5 F1-Score——Precision 和 Recall 的调和平均

```
           2 × Precision × Recall
F1 = ——————————————————————————————
            Precision + Recall
```

**为什么用调和平均而不是算术平均？**

```
Precision=0.9, Recall=0.1（玩赖模型：只说少数的确切答案，大部分漏掉）
  算术平均 = (0.9+0.1)/2 = 0.5  ← 看起来还行？
  调和平均 = 2×0.9×0.1/(0.9+0.1) = 0.18  ← 现原形了！

Precision=0.5, Recall=0.5（均衡模型）
  算术平均 = (0.5+0.5)/2 = 0.5
  调和平均 = 2×0.5×0.5/(0.5+0.5) = 0.5  ← 两者一致
```

调和平均的特性：**只要有一个很低，结果就拉低。** 它惩罚"偏科"，只奖励两个指标都高的模型。这就是为什么用 F1 而不是简单平均。

**你的良性 F1 = 0.94，恶性 F1 = 0.97**：两边的 Precision 和 Recall 都很高，F1 自然也很高。

### 5.6 Support（支持数）

**含义**：这个类别在测试集中有多少个样本。

```
良性 support = 41：测试集有 41 个良性样本
恶性 support = 73：测试集有 73 个恶性样本
总计 114 个
```

Support 告诉你：
- 每个指标的**可信度**——support 太小（比如只有 3 个），算出来的指标就不靠谱
- 数据是否**类别不平衡**——这里 73:41，恶性大约是良性的 1.8 倍，有轻微倾斜但不算严重

### 5.7 Accuracy（准确率）

```
        TP + TN         预测正确的数量
Acc = ——————————— = ——————————————————————
        TP+TN+FP+FN       总样本数

你的 Accuracy = 0.96：114 个样本中，预测对了约 109 个
```

Accuracy 是最直观的指标，但有坑——**类别极度不平衡时会骗人**。

```
极端例子：1000 个样本中，990 个良性，10 个恶性。
模型闭着眼全部判良性：
  Accuracy = 990/1000 = 99%  ← 看起来很美
  但恶性 Recall = 0/10 = 0%  ← 一个恶性都没抓住！
```

所以 Accuracy 必须结合 Precision、Recall 一起看。你的数据里类别不算极端不平衡，Accuracy = 0.96 是可信的。

### 5.8 Macro Avg（宏平均）vs Weighted Avg（加权平均）

**Macro Avg**：每个类别的指标单独算，然后直接取平均，不关心每个类别有多少样本。

```
macro_precision = (P_良性 + P_恶性) / 2 = (0.93 + 0.97) / 2 = 0.95
macro_recall    = (R_良性 + R_恶性) / 2 = (0.95 + 0.96) / 2 = 0.96
macro_f1        = (F1_良性 + F1_恶性) / 2 = (0.94 + 0.97) / 2 = 0.95
```

> 每个类别一视同仁，不管样本多少。适合关注"小类别也做得好不好"的场景。

**Weighted Avg**：每个类别的指标乘以该类的样本占比，再求和。

```
weighted_precision = (41/114)×0.93 + (73/114)×0.97 = 0.96
weighted_recall    = (41/114)×0.95 + (73/114)×0.96 = 0.96
weighted_f1        = (41/114)×0.94 + (73/114)×0.97 = 0.96
```

> 样本多的类别权重更大。适合关心"大多数样本预测得怎么样"的场景。

你的 macro avg ≈ weighted avg，说明两个类别表现都很均衡，没有哪个类别拖后腿。

### 5.10 这些指标和损失函数的关系

你可能会问：训练时用的是交叉熵损失，评估时用的却是 Precision/Recall/F1，它们之间是什么关系？

```
交叉熵损失（训练用）：
  - 直接可导，适合梯度下降
  - 对概率的"校准度"敏感——不仅关心对不对，还关心概率是否合理
  - 例如真实标签=1，预测概率=0.99 的损失远小于预测概率=0.51
    但两者在 0.5 阈值下都判对了（Accuracy 一样）

Precision / Recall / F1（评估用）：
  - 不可导，不能直接用作损失函数
  - 只关心"过没过阈值"，不关心概率的具体值
  - 直接回答业务问题：漏诊率多少？误判率多少？

用交叉熵训练 → 用 F1 评估，这是分类任务的标准范式
```
