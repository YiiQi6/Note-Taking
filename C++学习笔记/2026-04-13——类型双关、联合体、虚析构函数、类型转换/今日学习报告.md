## 一、类型双关（Type Punning）

### 概念

类型双关是指将一块内存按不同的类型来解释。在C++中，由于类型系统的存在，正常情况下不能直接将 `int*` 当作 `float*` 来用，但有时候我们需要绕过这个限制。

### Cherno强调的三种实现方式

#### 1. 指针强制转换（reinterpret_cast）

```cpp
Vec3 vector = {1.0f, 2.0f, 3.0f};
float* f = reinterpret_cast<float*>(&vector);
```

最直接的方式，但也是最危险的方式。

#### 2. 联合体（Union）

```cpp
union {
    float f;
    int i;
} converter;
converter.f = 1.0f;
```

通过共享内存实现类型双关。

#### 3. memcpy

```cpp
double d = 3.14159;
long long bits;
std::memcpy(&bits, &d, sizeof(double));
```

这是最安全的方式，因为 memcpy 是C++中唯一保证行为良好的类型双关方法。

### 重要概念：Strict Aliasing规则

Cherno特别强调，某些类型双关方式可能触犯C++的strict aliasing规则，导致未定义行为。编译器可能会优化掉某些代码，或者在不同平台上表现不一致。

### 实际应用场景

- 序列化数据
- 网络协议处理
- 直接操作二进制数据

## 二、联合体（Unions）

### 基本概念

联合体是一种特殊的数据结构，所有成员共享同一块内存。联合体的大小等于其最大成员的大小。

```cpp
struct Entity
{
    float x, y;
    union
    {
        int id;
        float score;
    };
};
```

当 `id` 和 `score` 不会同时使用时，可以用联合体节省内存。

### 关键特性

- **内存共享**：写入一个成员会覆盖其他成员的值
- **匿名联合体**：直接通过 `e.id` 或 `e.score` 访问，不需要 `e.data.id`
- **命名联合体**：需要通过 `e.data.id` 访问

### 应用场景

1. **内存复用**：不同字段不会同时使用
2. **类型双关**：利用共享内存特性
3. **变体类型**：模拟有多种类型的值

## 三、虚析构函数（Virtual Destructors）

### 为什么需要虚析构函数

这是Cherno在讲解继承时反复强调的重点！

```cpp
class Base
{
public:
    virtual ~Base() {}  // 虚析构函数
};

Base* ptr = new Derived();
delete ptr;  // 正确调用 Derived::~Derived() 然后 Base::~Base()
```

### 如果析构函数不是虚的

```cpp
class BaseBad
{
    ~BaseBad() {}  // 非虚！
};

BaseBad* ptr = new DerivedBad();
delete ptr;  // 只调用 BaseBad::~BaseBad()！
                 // DerivedBad::~DerivedBad() 永远不会被调用！
```

这会导致**严重的内存泄漏**！派生类的资源不会被正确清理。

### 原理：静态绑定 vs 动态绑定

为什么非虚析构函数会导致派生类析构函数不被调用？

- **非虚函数（包括析构函数）**：编译器采用**静态绑定**，通过指针类型在编译时确定调用哪个函数。因此 `Base*` 指针只会调用 `Base::~Base()`。

- **虚函数**：编译器采用**动态绑定**，通过虚指针（vptr）在运行时查找实际的类型。通过基类指针调用虚函数时，会根据对象实际类型调用对应的派生类函数。

虚析构函数的工作机制：编译器为每个包含虚函数的类创建**虚表（vtable）**，每个对象实例化时获得一个指向其类虚表的虚指针（vptr）。调用时通过 vptr 找到实际的析构函数。

### 多层继承中的虚析构函数

```cpp
class GrandBase { virtual ~GrandBase() {} };
class Middle : public GrandBase { ~Middle() override {} };
class Child : public Middle { ~Child() override {} };

GrandBase* ptr = new Child();
delete ptr;
// 正确的析构顺序：Child -> Middle -> GrandBase
```

## 四、类型转换（Type Casting）

### 隐式类型转换

隐式类型转换是编译器自动完成的类型转换，不需要程序员显式调用。Cherno指出，隐式转换虽然方便，但有时会产生意想不到的结果。

#### 常见的隐式转换场景

```cpp
// 1. 算术转换（宽类型提升）
int i = 10;
double d = i;  // int -> double（精度提升）

// 2. 精度损失
double d2 = 3.14;
int i2 = d2;   // double -> int（小数部分被截断）

// 3. 指针向上转型（派生类到基类）
class Base {};
class Derived : public Base {};
Derived d;
Base* ptr = &d;  // 隐式转换
```

#### 隐式转换的问题

```cpp
void Print(int value) {}

double d = 3.14;
Print(d);  // 警告：隐式转换可能丢失精度！
```

### 显示类型转换

C++提供了四种显式类型转换操作符，应优先使用这些而不是C风格转换。

#### 1. static_cast

```cpp
double d = 3.14;
int i = static_cast<int>(d);  // double -> int

Derived d;
Base* b = static_cast<Base*>(&d);  // 向上转型，总是安全
```

**编译时检查类型转换**，无法转换类型会在编译时报错。

#### 2. reinterpret_cast

```cpp
int value = 65;
char* c = reinterpret_cast<char*>(&value);  // 重新解释位模式
```

**无类型转换检查**，常用于类型双关。

#### 3. const_cast

```cpp
const int c = 100;
int* p = const_cast<int*>(&c);  // 移除 const
//const int* p2 = const_cast<int*>(&c); 不要这样使用
```

添加或移除 const/volatile 属性。只有当实际引用的对象不是常量时，才应该使用const_cast，否则会出错。

#### 4. dynamic_cast

```cpp
Base* b = new Derived();
Derived* d = dynamic_cast<Derived*>(b);  // 向下转型，失败返回nullptr
```

**运行时检查类型转换**，常用于安全的向下转型。失败时会返回 `nullptr`（指针）或抛出 `std::bad_cast`（引用）。

| 转换类型               | 安全性 | 使用建议        |
| ------------------ | --- | ----------- |
| `static_cast`      | 高   | 基本类型转换、向上转型 |
| `dynamic_cast`     | 高   | 向下转型，带检查    |
| `const_cast`       | 中   | 不得已时使用      |
| `reinterpret_cast` | 低   | 最后手段        |

**优先使用 static_cast，只有在需要向下转型时才用 dynamic_cast。**
